# Клауд Ферма (claude-farm)

Игрушечный по форме, но честный по механике оркестратор задач. Задача проходит через «дом» из четырёх комнат-зон, как изделие по конвейеру: в каждой комнате её обрабатывает пара агентов — **Driver** (делает работу) и **Tester** (проверяет результат). За ходом дела можно наблюдать в реальном времени на пиксель-арт дашборде.

Без единой npm-зависимости: только встроенные модули Node.js (v25, ES modules).

## Концепция

Дом — это конвейер. Каждая зона отвечает за свой этап жизненного цикла задачи:

- **Driver** в зоне выполняет работу (парсит, преобразует, прогоняет, архивирует);
- **Tester** в той же зоне придирчиво проверяет результат и выносит вердикт: «ОК» — задача идёт дальше, «БАГ» — задача отправляется назад (bounce);
- сами агенты по дому не ходят. **Задачи между зонами носит только Главный Агент** — оркестратор (`src/orchestrator.mjs`). Он один знает маршрут, считает попытки, объявляет события и решает, куда нести задачу после вердикта тестера.

Каждый шаг порождает событие (event bus), события пишутся в JSONL-лог, транслируются по SSE на дашборд и озвучиваются в консоли.

## Зоны и персонажи

| Порядок | Зона | Название | Driver | Tester | Что происходит |
|---|---|---|---|---|---|
| 1 | `kitchen` | Кухня — Сборка | The Scraper | The Cleaner | Парсинг входных данных в строки, чистка |
| 2 | `corridor` | Коридор — Обработка | The Editor | The Validator | Преобразование в CSV, валидация дат |
| 3 | `living` | Гостиная — QA | The Runner | The Sniffer | Прогон «глазами пользователя», поиск дублей и пустых полей |
| 4 | `bath` | Ванная — Релиз | The Archiver | The Sign-Off | Запись result.csv и manifest.json, финальная приёмка |

Когда Sign-Off доволен, в логе появляется: «Клиенту можно отправлять!»

## Режим Главного Агента (Claude)

Ферма умеет работать не только на встроенных sim-агентах: внешний оркестратор — например, Claude Code в роли Главного Агента — выполняет работу зон своими субагентами и транслирует каждый шаг на дашборд через `POST /api/event`:

```bash
curl -s -X POST localhost:8787/api/event \
  -d '{"type":"zone.enter","taskId":"boss-1","zone":"kitchen","message":"Задача переходит в зону «Кухня — Сборка»"}'
```

Допустимые `type`: `task.created`, `zone.enter`, `driver.start`, `driver.done`, `tester.start`, `tester.ok`, `tester.bounce`, `task.done`, `task.failed`. Дашборд отображает внешние события точно так же, как события встроенной фермы: токен ходит по зонам, прищепки двигаются, лента пишется.

## Быстрый старт

```bash
# Прогнать демо-задачу в консоли (с повествованием на русском)
npm run demo

# Поднять дашборд и смотреть на конвейер вживую
npm run serve
# затем открыть http://localhost:8787 и нажать «Демо»

# Своя задача из файла
node src/farm.mjs run "Список гостей" --input path/to/file.txt
```

## Как работает баунс-бэк: дата 35.12.2023

В `demo/demo-task.txt` строка 13 испорчена нарочно: `Игорь;35.12.2023`. Декабрь заканчивается 31-м числом, так что дальше происходит вот что:

1. **Кухня.** The Scraper разбирает файл на строки, The Cleaner подтверждает: строки есть, можно работать.
2. **Коридор.** The Editor собирает CSV. The Validator проверяет каждую дату как реальную календарную и спотыкается о `35.12.2023`. Он формирует заявку на исправление (день обрезается до последнего дня месяца: `35` → `31`) и выносит вердикт: `{ok: false, reason: «Дата 35.12.2023 не существует», bounceTo: "kitchen"}`.
3. **Баунс.** Главный Агент увеличивает счётчик попыток и несёт задачу обратно на кухню. The Scraper пересобирает данные уже с исправлением — в логе появляется «Паша исправил дату».
4. **Второй заход.** Коридор: Validator доволен. Гостиная: The Runner открывает CSV как пользователь, The Sniffer не находит ни дублей, ни пустых полей. Ванная: The Archiver пишет `output/<taskId>/result.csv` и `manifest.json`, The Sign-Off проверяет, что оба файла на месте и не пустые: «Клиенту можно отправлять!»

Если тестеры заворачивают задачу слишком упорно и число попыток превышает `maxAttempts` (по умолчанию 3 в `farm.config.json`), Главный Агент объявляет `task.failed`, и задача снимается с конвейера.

## Экспериментальный real-режим (claude CLI)

Вместо детерминированных sim-агентов в комнатах могут работать настоящие модели — через локально установленный `claude` CLI:

```bash
node src/farm.mjs run "Список гостей" --input demo/demo-task.txt --real
```

Каждый агент получает промпт со своей ролью, задачей зоны и сводкой накопленных данных (`claude -p "..."` с моделью из `farm.config.json`). Тестеры обязаны начинать ответ со слова **ОК** или **БАГ**. Режим экспериментальный: ответы моделей недетерминированы, тесты его не используют.

## Доступность

Дашборд собран по чек-листу WCAG AA, чек-лист внедрён: семантическая разметка и состояния зон дублируются текстом и глифом (не только цветом), лента событий — live-регион (`role="log"`), статусные объявления коалесцируются (`role="status"`, не чаще раза в 2 секунды), прогресс отмечен `aria-current="step"`, анимации уважают `prefers-reduced-motion`, мигание при ошибке ограничено тремя вспышками в секунду, звук по умолчанию выключен и управляется кнопкой с `aria-pressed`.
